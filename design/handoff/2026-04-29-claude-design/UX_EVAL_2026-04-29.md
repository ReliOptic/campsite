## 3. 합의된 P0 차단 사유

4인 전문가가 독립적으로 분석했음에도 다음 5개 항목은 **공통적으로 P0(즉시 반려)** 으로 판정되었다.

### 3.1 P0-1 · 페르소나-시나리오 불일치 (analyst)

영웅 시안 F-2가 `payments-v1` 단일 시나리오(Stripe PCI-DSS 보안 검토)로 고착되어 있다. 발주서 §3.1이 정의한 페르소나 희원(32세, 비코더 친화, 30분 사이드 프로젝트)의 작업 범주가 아니다. 발주서 §10.3의 5인 페르소나 시험을 **본 시안만으로는 통과 불가**.

→ **권고**: `copy-active-camp.md:317-319`의 시나리오 D(로그인 페이지)를 정식 변형으로 추가 산출.

### 3.2 P0-2 · §6 산출물 70% 미수령 (critic)

| 미수령 산출물 | 발주서 카테고리 |
|---|---|
| PNG-8/24 sprite sheet | A 화로 자산 |
| 5상태 × 4프레임 동물 sprite (총 120프레임) | B 참여자 |
| 환경 PNG 6점 | C 환경 |
| 768px 폴백 시안 | F 화면 시안 |
| manifest.json + SHA-256 | H 매니페스트 |
| Figma·Penpot 원본 | H 인계 자료 |
| 사용 가이드 PDF | H 인계 자료 |

수령품은 발주서 §11 M2 1차분 LoFi prototype에 해당. **PNG 자산이 단 한 점도 없다**. CSS clip-path + SVG `data:` URI는 §6.1 명세("PNG-8/24 ... 합산 600KB 이하")의 대체재가 아니다.

→ **권고**: 본 핸드오프 위상을 "M2 1차분 LoFi prototype"으로 재정의. M3~M5 산출물(PNG 자산·폴백·매니페스트) 후속 발주 필수.

### 3.3 P0-3 · §8 절대 금지 4항 위반 (critic)

발주서 §8 "절대 금지 사항" 10항 중 다음 4항이 코드상 명백히 위반:

| 위반 항 | 발견 위치 |
|---|---|
| §8-2 둥근 모서리(>0px) | `scene.css:471` `.participant .glow { border-radius: 50% }` |
| §8-5 픽셀 그리드 위반·블러 금지 | `scene.css:201, 373, 472, 645` 다중 `blur(8~28px)` 적용 |
| §8-8 텍스트 절단 | `active-camp.jsx:238, 242` 32자·24자 슬라이스 이중 적용 |
| §8-1 1px 실선 | `scene.css:120` `outline: 1px solid rgba(0,220,229,0.2)` |

INTEGRATION.md §3.3은 "10항 중 9항 완전 준수"로 자평하였으나 **자기 검증 신뢰도가 결정적으로 손상**되었다.

→ **권고**: 시각 자산 채택 시 위 4항 우선 보정. 통합 시 INTEGRATION.md의 자기 검증 결과를 권위 자료로 사용 금지.

### 3.4 P0-4 · 접근성 6항 미준수 (code-reviewer)

WCAG 2.2 AA 기준 위반:

| WCAG 기준 | 위반 |
|---|---|
| 2.1.1 키보드 | 클릭 가능한 `<div>`(term-station, participant, item, mini-camp)에 `role="button"`·`tabIndex`·Enter/Space 핸들러 부재 |
| 2.4.3 포커스 순서 | Drawer focus trap 부재, ESC 키 핸들러 부재 |
| 4.1.2 이름·역할·값 | Drawer에 `role="dialog"`·`aria-modal`·`aria-labelledby` 부재 |
| 1.4.1 색상 비의존 | state-pip이 색상에만 의존 (형태·아이콘 동반 부재) |
| 1.4.3 텍스트 콘트라스트 | `--on-surface-faint #6c6877` on `--surface #131318` ≈ 3.4:1 (AA 미달) |
| 1.1.1 비텍스트 콘텐츠 | 동물 sprite·flame 등 의미 전달 시각 요소가 alt 부재 |

→ **권고**: 시각 자산은 채택 가능, **인터랙션 코드는 폐기 후 재작성**. INTEGRATION.md §5.2 Step 4 vanilla JS 변환 시점에 일괄 수정.

### 3.5 P0-5 · 외부 의존 위험 (critic + code-reviewer 공동)

- React 18 + Babel CDN runtime: 발주서 §5.7 CSP `unsafe-eval` 요구 + 초기 페인트 지연
- Google Fonts CDN: 인터넷 없으면 한글 위계 붕괴 (Pretendard fallback chain은 system-ui)
- Babel `text/babel` 컴파일: production 부적합

→ **권고**: 본체 통합 시 React/Babel/CDN 일체 제거. INTEGRATION.md §6.4 "그대로 가져오지 않는 것"과 일치.

## 4. P1 강력 권고 사항

### 4.1 P1-1 · Left Terminal Rail 정보 밀도 과적 (analyst)

LOCAL 7행 + Terminal Stations 4 카드(각 3행) = 19행 monospace 메타가 240px 폭에 적층. 발주서 §3.2 희원 정서("어렵지 않다, 복잡하지 않다")와 충돌.

→ 첫 진입 4행 표시 + 펼치기 토글로 압축. 5초 회복 가능성 확보.

### 4.2 P1-2 · 화로 z-index 위계 역전 (designer)

`mission-card z:4` > `hearth z:3`. 발주서 §4.3 "화로가 시각적 주인공" 원칙에 어긋남. 화로 96×96px가 1440px scene 높이의 12%로 시각 무게 부족.

→ 화로 z-index 상승 + 미션 카드는 화로 옆 또는 우측 재배치. 화로 자체 크기를 1.5배 확대.

### 4.3 P1-3 · 한·영 병기 위계 규칙 부재 (analyst)

Mission Card·Return Panel 모든 항목이 영문+한국어 동시 노출. 길이 가이드는 있으나 두 줄 적층의 위계 규칙 부재. ReturnPanel 한 항목이 4행 차지로 §10.2 768px 검수 불가.

→ 1차 정보(Mission Title, headline, Next move 본문)는 동시 노출, 2차 정보는 사용자 언어 단일 + hover 보조 언어로 전환.

### 4.4 P1-4 · Canyon 테마 콘트라스트 미달 (code-reviewer)

`--on-surface-faint #9a826b` on `--surface-low #d6c0a3` ≈ 2.6:1.

→ Canyon 보조 텍스트 휘도 재조정.

### 4.5 P1-5 · Dave the Diver 도달도 3/10 (designer)

CSS clip-path + box-shadow + SVG 16×16 단일 프레임은 발주서 §4.4 "HD 픽셀 아트(데이브 더 다이버)" 약속의 7단계 미달.

→ 카테고리 A·B PNG sprite 후속 발주 최우선. M3 라운드의 결정적 항목.

## 7. 핵심 통찰 (4인 합의)

본 핸드오프는 의미론적 기반(5종 화로 색채, 가족룩 어휘, No-Line, IA 골격)은 v1.0 채택 가능 수준이나, 다음 4개 차원에서 출시 차단 사유가 명백하다.

1. **산출물 충족도** (3/10) — M2 본격 산출물의 70%가 미수령
2. **접근성** (3/10) — WCAG 2.2 AA 6항 위반
3. **Dave the Diver 도달도** (3/10) — HD 픽셀 아트 약속의 7단계 미달
4. **참고 좌표 도달도** (3/10) — 시각 정체성 60% 수준

**4인 모두 v1.0 직행 부적격을 합의**한다. 시각 자산(tokens·일부 CSS·모션 사양)은 채택 가능, **인터랙션 코드는 폐기 후 재작성**, **PNG 자산·폴백 시안·a11y는 다음 라운드 발주 필수**.
