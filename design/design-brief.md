# Campsite 제품 디자인 발주서 (Design Brief)

> 발주처: ReliOptic / Kiwon Cho (ckiwon7@gmail.com)
> 프로젝트: Campsite — Recovery-First AI Workspace
> 문서 버전: v1.0
> 작성일: 2026-04-29
> 회수 의뢰처: GitHub https://github.com/ReliOptic/campsite
> 발주 범위: 제품 시각 정체성(Visual Identity), 픽셀 아트 자산군, Camp View 화면 디자인, Family-Look 가이드 정합

본 문서는 외부 디자인 전문가(또는 스튜디오)에게 Campsite 제품 디자인 일체를 의뢰하기 위한 단일 참조서이다. 본 문서로 견적 산정·작업 착수·산출물 검수가 가능하도록 자기 완결적으로 작성되었다.

---

## 1. 프로젝트 개요

### 1.1 한 문장 정의

Campsite는 다수의 프로젝트와 AI 에이전트를 하나의 시각적 캠프 공간에 모아, 사용자가 자리를 비웠다 돌아왔을 때 5초 안에 모든 상황을 파악하고 다시 작업을 재개할 수 있도록 돕는 **recovery-first AI workspace**이다.

### 1.2 제품 형태

- 명령행 도구(CLI)와 브라우저 기반 시각 표면(Camp View)으로 구성된 이중 표면(dual-surface) 제품
- 명령행 표면(Focus mode)과 시각 표면(Camp mode)이 동일한 어휘·동일한 의미론·동일한 정서를 공유하도록 설계된다
- 본 발주의 주된 대상은 **시각 표면(Camp mode)** 이며, 명령행 표면의 색채 의미론과 정합하도록 한다

### 1.3 본 발주의 위상

- Campsite v1.0 정식 출시 직전 단계의 시각 정체성 확립을 목적으로 한다
- 출시 일정: 2026년 9월 (단, 자산 산출 시점에 따라 ±4주 변동 허용)
- 본 발주가 v1.0 디자인 동결의 결정적 조건이다

## 2. 사업적 맥락 및 발주 목적

### 2.1 현재 상태

- 기능 구현은 v0.2.0 시점에서 통합 테스트 223건 통과로 안정화되어 있다
- 정적 SVG·CSS 기반의 임시 시각 표면이 작동 중이다
- 픽셀 아트 자산은 단 한 점도 산출되지 않았다(`design/export/`가 비어 있다)
- HTML 폴백 모드는 유지하되, 본 발주를 통해 **고품질 픽셀 아트 storyworld**로 전환한다

### 2.2 발주 목적

다음 세 가지 결손을 해소한다.

1. **시각 정체성 부재**  현재의 임시 표면은 정서를 전달하지 못한다
2. **자산 미생산**  Phaser 기반 storyworld(`camp-client/`)가 자산 부재로 가동 불가 상태이다
3. **페르소나 검증 불가**  비코더 30대 사용자에 대한 정성 검증이 시각 정체성 부재로 진행되지 못하고 있다

### 2.3 본 발주가 산출해야 할 효익

- 사용자가 첫 진입 30초 안에 "차분하고 야간적인 작업 공간"이라는 정서를 인지한다
- Camp View가 단순 일러스트가 아닌 **정보 우선 대시보드**로 기능한다
- 시각 위계(불 > 미션 > 참여자 > 배경)가 명확히 작동한다
- 5종 화로 상태(불씨·모닥불·등불·연기·장작)가 시각만으로 의미 분별이 가능하다

## 3. 사용자 페르소나

### 3.1 일차 페르소나: 희원 (32세, 프론트엔드 개발자)

- 회사에서 3개 프로젝트를 동시에 진행 중
- 바이브코딩 경험이 없으며, 코딩 자체를 좋아하지는 않으나 결과물의 완성도는 중시한다
- 터미널 환경에 거부감이 있고 AI 도구의 다양성에 피로를 느낀다
- 퇴근 후 30분의 사이드 프로젝트 시간을 갖고 싶으나 매번 "어디까지 했더라" 단계에서 10분을 소진한다

### 3.2 희원이 발주된 디자인에서 기대하는 정서

- "어렵지 않다, 복잡하지 않다, 쉽다"는 인상
- 열었을 때 "예쁘다, 차분하다"는 첫 반응
- 다음 행동이 즉시 보이는 정보 전달성
- 스트레스 없이 생산적이라는 감각

### 3.3 반(反) 페르소나(피해야 할 사용자상)

- 터미널 최적화에 집착하는 power user (별도 도구 OMC가 적합)
- 100개 에이전트 동시 운용을 원하는 orchestration builder
- 과도한 도파민 자극과 게이미피케이션을 선호하는 사용자

## 4. 디자인 목표 및 핵심 정서

### 4.1 추구해야 할 정서

| 정서 축 | 내용 |
|---|---|
| Tranquil (정온) | 야간의 캠프, 낮은 도파민, 명상적 |
| Operational (실무적) | 단순 일러스트가 아닌 작동하는 대시보드 |
| Warm (따뜻함) | 사이버틱 차가움이 아닌 인간적 온기 |
| Sharp (정연함) | 부드러운 모서리·둥근 형태 회피, 픽셀 격자 엄수 |
| Wide-breath (긴 호흡) | 단발 스프린트가 아닌 장기 창작 곡선 지원 |

### 4.2 절대 추구하지 말아야 할 정서

- 기업용 대시보드(Salesforce·Jira류)
- 모바일 게임의 카툰 미감
- 판타지 RPG·중세 메디벌
- 아동용 그래픽
- 과장된 긴급성·알림 폭격
- 남성 개발자 중심의 전형적 터미널 미학(녹색 단색 코드 스크롤 등)

### 4.3 시각 위계(Visual Hierarchy) — 절대 원칙

화면 어느 시점에서도 다음 순서가 보존되어야 한다.

1. **화로(fire-state)가 시각적 주인공**  화면 중심, 가장 큰 시각 무게, 상태별 색·크기·애니메이션 분별
2. **미션·다음 행동(mission, next-move)이 즉시 가독**  카드 패널, 충분한 자간·행간, 절단 없음
3. **참여자(participant)는 스캔 가능한 보조 정보**
4. **배경(environment)은 분위기만 제공**  정보를 절대 방해하지 않음

### 4.4 그래픽 품질 기준

본 발주는 다음 수준을 요구한다.

- **HD 픽셀 아트**  저해상도 레트로 8-bit가 아닌 고해상도 정연 픽셀 아트(참고: 「데이브 더 다이버(Dave the Diver)」)
- **풍부한 라이팅**  캠프파이어 광원이 주변 객체에 실제로 반사되는 림 라이팅·앰비언트 오클루전
- **디테일 패럴랙스 배경**  원경 산·중경 나무 실루엣이 깊이감 있게 적층
- **일관된 픽셀 그리드**  캐릭터·배경·파티클이 동일한 픽셀 밀도(혼합 금지)
- **블러 절대 금지**  파티클을 포함한 모든 시각 요소는 픽셀 격자에 스냅된다

## 5. 디자인 시스템 — 사전 확정 사항

본 항목은 발주 이전에 확정된 사항이며, 변경 제안은 별도 협의 후에만 가능하다.

### 5.1 색채(Palette)

#### 표면(Surface) 위계

| 토큰 | HEX | 용도 |
|---|---|---|
| Background | `#131318` | 가장 깊은 야간 표면 |
| Surface Container Low | `#1b1b20` | 대형 구조 블록 |
| Surface Container High | `#2a292f` | 인터랙티브 카드, 부유 패널 |
| Surface Container Highest | `#35343a` | 카드 상단 강조 |

표면 위계는 **선(line)이 아닌 명도 변화**로 구분한다. 1px 실선 분할은 금지이다.

#### 의미 색채(Semantic Colors)

| 토큰 | 용도 | HEX 권장 |
|---|---|---|
| Primary (Moonlight Lavender) | 고차 정보·인터랙티브 상태 | `#cdc1e0` |
| Secondary (Moonlight Blue) | 보조 정보 | `#bac8dc` |
| Tertiary (Neon Cyan) | 핵심 CTA·활성 상태(절제 사용) | `#00dce5` |
| On-Surface | 본문 텍스트 | `#e4e1e9` |
| Outline (Ghost) | 절대 필요한 경우의 미세 외곽선 | `#49454e` (불투명도 20%) |

#### 화로 상태(Fire-State) 의미 색채

| 상태 | 의미 | 색채 영역 |
|---|---|---|
| 불씨 (bulssi) | 막 시작됨, 약한 신뢰 | 약한 ember orange |
| 모닥불 (modakbul) | 활발한 진행 | 강한 ember orange + warm glow |
| 등불 (deungbul) | 검토 대기 | lantern gold |
| 연기 (yeongi) | 차단·도움 필요 | smoke blue-gray + faint cyan edge |
| 장작 (jangjak) | 다음 행동 준비 | ready green (절제) |

화로 색채 의미는 **시스템 의미론을 침해할 수 없다**. 환경 테마(오로라 북, 그라나이트 포레스트, 캐년 데이라이트)에 따라 배경 색조가 변할 수 있으나, 5종 상태 색채는 항상 가독·일관해야 한다.

### 5.2 타이포그래피

| 등급 | 글꼴 | 용도 |
|---|---|---|
| Display / Headline | Space Grotesk | 영웅 영역, 섹션 헤더 |
| Technical Label | Space Grotesk (small + uppercase + letter-spacing 0.05rem) | 메타데이터·시스템 상태 |
| Body | Inter | 본문 |
| 한국어 동반 | 시스템 폰트(노토 산스 KR 권장) | 화로 상태명 등 한국어 표기 |

순수 흰색(#FFFFFF) 사용 금지, 본문 최대 휘도는 `#e4e1e9`이다.

### 5.3 모서리(Corner Radius)

- **모든 모서리는 0px(엄수)**
- 곡면은 디지털 세계관에 부적합한 유기적 인상을 준다는 사전 결정에 따른다
- 단, 픽셀 아트의 자연 외곽선(나무 실루엣 등)은 예외이다

### 5.4 그림자(Shadow)

- 통상적인 어두운 그림자는 사용하지 않는다
- 입체감은 표면 위계의 명도 차로 표현한다
- 부유 요소(모달 등)에는 `on-surface` 5% + 32px blur의 매우 옅은 발광 그림자만 허용한다

### 5.5 코너 액센트(Corner Accents)

- 4×4 픽셀 단위의 tertiary 색상 모서리 액센트는 사용 가능
- 픽셀 아트 정체성 환기를 위한 장식이며, 정보 분리 목적으로 사용하지 않는다

### 5.6 간격(Spacing)

- 섹션 간에는 **충분한 여백**을 둔다(터미널의 압축미와 캠프의 공간감을 동시에 보존)
- 비대칭 마진을 적극 활용한다(헤드라인과 본문의 좌측 시작점이 달라도 무방)

### 5.7 동작(Motion)

- 모든 모션은 정온하다  과격한 변화·진동·번쩍임 금지
- `prefers-reduced-motion` 환경에서는 모든 애니메이션이 차단되어야 한다
- 캠프파이어 깜빡임은 부드러운 사인파, 주기 1.5~2.5초
- 활성 참여자(모닥불)는 미세한 펄스(주기 3초 이상)
- 별·달 등 환경 요소는 정적이거나 매우 느린 페이드만 허용

## 6. 산출물 명세(Deliverables)

본 발주의 핵심이다. 모든 자산은 본 절의 명세를 따른다.

### 6.1 공통 기술 사양

| 항목 | 사양 |
|---|---|
| 캔버스 기준 | 480×320 px (정수 배율 스케일링) |
| 픽셀 그리드 | 동일 밀도(자산 간 혼합 금지) |
| 색채 팔레트 | 16색 이내 권장(상한 24색) |
| 파일 형식 | PNG-8 또는 PNG-24 (투명 배경) |
| 자산 합산 용량 | 600KB 이하 |
| 명명 규약 | `<category>-<role>-v<n>.png` 예: `campfire-active-v1.png` |
| 매니페스트 | `design/export/manifest.json` 자동 생성용 메타데이터 동봉 |

### 6.2 카테고리 A — 화로(Campfire) 자산 6점

| ID | 자산 | 크기(px) | 용도 |
|---|---|---|---|
| A-1 | Campfire Core | 64×64 | 메인 미션 카드 중심 불 |
| A-2 | Bulssi (불씨) | 32×32 | 막 시작된 약한 불꽃 |
| A-3 | Modakbul (모닥불) | 48×48 | 활발하게 타오르는 불 |
| A-4 | Deungbul (등불) | 32×48 | 검토 대기, 따뜻한 등 |
| A-5 | Yeongi (연기) | 32×64 | 막힌 상태의 연기 신호 |
| A-6 | Jangjak (장작) | 48×32 | 쌓인 장작, 준비된 다음 행동 |

각 자산은 **정적 PNG** 1점 + (선택) 4프레임 애니메이션 sprite sheet 1점.

### 6.3 카테고리 B — 참여자(Participant) 스프라이트 시트

프로젝트는 1마리의 동물 캐릭터를 갖는다(여우·토끼·올빼미·고양이·곰·사슴 6종).

| ID | 자산 | 크기(px) | 비고 |
|---|---|---|---|
| B-1~B-6 | 6종 동물 sprite sheet | 16×16 px 단위 | 5상태 × 4프레임 = 동물 1종당 20프레임 |

- 시각 위계상 16×16 px 이하의 **마이크로 스프라이트**로만 존재한다
- 동물의 표정·자세는 화로 상태와 매핑된다
  - bulssi: 조심스러운 자세
  - modakbul: 집중한 자세
  - deungbul: 의자에 앉아 자료를 보는 자세
  - yeongi: 정지·도움 요청 자세
  - jangjak: 다음 작업을 준비하는 자세

| ID | 자산 | 크기(px) | 용도 |

| C-6 | Camp Grounds Detail | 480×64 | 중경 캠프 부지 |

| ID | 자산 | 크기(px) | 용도 |
|---|---|---|---|
| D-1 | Corner Accents Set | 16×16 | 패널 모서리 4종(좌상·우상·좌하·우하) |

### 6.6 카테고리 E — 환경 테마 응용 팔레트(선택)

기본 야간 테마 외에 다음 3종의 환경 테마 팔레트를 부수 산출한다(자산 본체 재제작 불필요, 색조 시프트 가이드만 제공해도 무방).

## 8. 절대 금지 사항(Hard Constraints)

다음은 사양 위반이며 검수 단계에서 즉시 반려된다.

1. 1px 실선 외곽선·구분선 사용
2. 둥근 모서리(border-radius > 0)
3. 표준 어두운 드롭 섀도
4. 카툰풍·이모지·과장된 표정
5. 픽셀 그리드 위반(블러·반투명 모서리)
6. 화로 상태 색채를 시스템 외 의미로 차용(예: tertiary 색을 경고에 사용)
7. skeuomorphic 메타포(나무 표지판·종이 메모지 등)로 운영 정보 전달
8. 텍스트 절단(ellipsis가 본문 정보에 적용)
9. 100% 순백(#FFFFFF) 텍스트
10. 일러스트가 화면을 점유하고 그 위에 UI를 후처리로 얹는 방식 — UI 레이어링이 우선이다

## 10. 검수 기준(Acceptance Criteria)

각 검수는 객관적 통과 조건을 갖는다.

### 10.1 자산 단위 검수

- [ ] 모든 자산이 명시된 픽셀 그리드와 색채 팔레트를 따른다
- [ ] 5종 화로 상태가 시각만으로 의미 분별이 가능하다(블라인드 테스트로 검증)
- [ ] 자산 합산 용량이 600KB 이하이다
- [ ] 모든 PNG는 알파 채널 정합·반(半)투명 픽셀 무 검증을 통과한다
- [ ] manifest.json이 모든 자산을 SHA-256 해시와 함께 등록한다

### 10.2 화면 시안 검수

- [ ] 1440×900 px 시안에서 시각 위계(불 > 미션 > 참여자 > 배경)가 보존된다
- [ ] 768 px 폴백 시안에서도 영웅 영역이 깨지지 않는다
- [ ] 모든 텍스트가 절단 없이 가독한다
- [ ] 모든 텍스트가 한국어·영어 실 카피로 채워진다(샘플 텍스트 금지)
- [ ] 5초 이내 복귀 컨텍스트 회복이 가능하다(타이머 측정으로 검증)

### 10.3 페르소나 검증

- [ ] 5인 페르소나 시험에서 4인 이상이 30분 이내 첫 캠프 진입 성공
- [ ] 정성 인터뷰에서 "예쁘다", "차분하다", "어렵지 않다"라는 키워드가 다수 등장
- [ ] 도파민·자극·게임 같다는 키워드가 다수 등장하지 않는다

### 10.4 시스템 정합 검증

- [ ] 모든 자산이 `prefers-reduced-motion` 환경에서 정상 표시된다
- [ ] 환경 테마 3종(오로라·그라나이트·캐년) 적용 시 화로 상태 색채 의미가 침해되지 않는다
- [ ] Camp mode와 Focus mode(터미널 출력)의 어휘·정서가 동일한 제품으로 인식된다

### 10.5 인계 자료 완결성

- [ ] Figma·Penpot 원본이 편집 가능 상태로 인계된다
- [ ] 8~12페이지 분량의 사용 가이드 PDF가 제공된다
- [ ] design-tokens.json이 색·간격·타이포·모션 토큰을 머신 리더블 형태로 제공한다
