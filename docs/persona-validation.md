## 1. 목적과 위상

본 프로토콜은 발주서 §10.3과 로드맵 `docs/completion-roadmap.md` Phase 11이 요구하는 5인 페르소나 시험의 단일 진실원이다. 시험 통과는 v1.0 동결의 결정적 조건이며, 미통과 시 v1.0 출시는 차단된다.

**본 시험은 정성 신호 탐지(qualitative signal detection)가 목적이며 정량 일반화가 아니다.** 5인 표본은 통계적 신뢰구간이 넓으므로 통과 기준 4/5는 "통계적 유의성"이 아닌 "정성적 신호 임계값"이다. 본 위상은 §7 통과 기준의 의미를 오인하지 않도록 §1에 사전 명시한다.

본 프로토콜은 v1.0 동결을 위한 1차 시험만 관할한다. 출시 후 지속 시험은 별도 문서(`docs/post-release-validation.md`, 추후 작성)로 분리한다.

## 2. 페르소나 정의

### 2.1 기본 페르소나

PRD §3.1 그대로 인용:

- 희원 (32세, 프론트엔드 개발자, 비코더 친화)
- 회사에서 3개 프로젝트 동시 진행, 바이브코딩 무경험
- 코딩 자체를 좋아하지는 않으나 결과물 완성도는 중시
- 터미널 환경 거부감, AI 도구 다양성 피로
- 퇴근 후 30분 사이드 프로젝트 시간을 갖고 싶으나 매번 "어디까지 했더라" 단계에서 10분 소진

### 2.2 5인 시험자 모집 기준

다음 조건을 모두 충족하는 시험자만 채택:

- 30~40대
- 코딩 경험 0~3년 또는 비코더
- 도구 호기심 (사이드 프로젝트 1회 이상 시도)
- 한국어 모어 사용자
- macOS·Windows·Linux 사용자 (ChromeOS·WSL 비대상)
- 사전 Campsite 노출 0건 (자가 보고 검증)

### 2.3 반(反) 페르소나 차단

다음에 해당하는 응시자는 모집 단계에서 제외:

- OMC 사용자 (orchestration builder 성향)
- 100개 에이전트 동시 운용 욕구를 가진 power user
- 터미널 최적화에 집착하는 사용자

자가 보고 설문(§13 부록)으로 사전 차단한다.

### 2.4 모집 채널

다음 두 채널 병행하여 편향을 분산:

- 발주처 개인 네트워크 (1~2명, 페르소나 적합성 높음)
- 디자인·코딩 커뮤니티 공고 (3~4명, 다양성 확보)

각 채널의 시험자 비율은 모집 결과 보고서에 명시한다.

## 3. 시험 환경

| 항목 | 내용 |
|---|---|
| 기기 | 사용자 본인 노트북 (다양성 확보) |
| OS | macOS·Windows·Linux 혼합 모집 (각 OS 1~2명) |
| 인터넷 | 첫 시험 인터넷 연결, 두 번째 시험 오프라인 (NFR §5.7 검증) |
| 시간 | 1인당 60~90분 (시범 + 자유 사용 + 인터뷰) |
| 녹화 | 화면 + 음성 + 표정 (사전 동의 후) |
| 백업 | LTE 핫스팟 권고 (인터넷 단절 대비) |

시험 환경은 시험자 본인 노트북을 사용함으로써 실제 사용 맥락에 가까운 정성 신호를 회수한다. 표준화된 시험 기기 사용은 본 프로토콜에서 비목표이다.

## 4. 시험 시나리오 4종

각 시험자가 모두 수행한다.

### 4.1 T1 — 첫 진입

설치부터 첫 캠프 화면 도달까지 자기 주도. 시험자는 다음 명령을 자기 의지로 입력한다.

```
curl -fsSL https://raw.githubusercontent.com/ReliOptic/campsite/main/install.sh | bash
campsite setup
campsite camp
```

목표: 10분 이내 첫 캠프 화면 도달.

### 4.2 T2 — 첫 미션 설정

T1 직후 "로그인 페이지 만들기" 미션을 만들고 화로를 점화한다.

목표: 5분 이내 미션 설정 + 첫 화로 점화 완료.

### 4.3 T3 — 일시 종료 후 복귀

T2 직후 캠프를 종료하고 30분간 본인의 평소 작업 또는 휴식을 한다. 단 시험 환경(노트북)은 30분간 유지하고, 다른 도구로 캠프 정보를 별도 기록하지 않는다. 30분 후 다시 캠프를 열어 5초 회복 측정을 수행한다.

목표: 5초 회복 성공률 80%.

### 4.4 T4 — 차단 시나리오

의도적 입력 오류를 통해 yeongi 표현 인지 여부를 측정한다. 차단 방법은 미션명에 환경 변수 누락을 시뮬레이션하는 입력(예: `${MISSING_VAR}` 같은 보간되지 않는 값을 미션명에 포함)을 시험자에게 안내한다. 시험자가 우회할 수 없도록 설계한다.

목표: yeongi 어휘 인지 + "막힌 상태" 의미 파악.

## 5. 측정 지표 5종

각 지표는 측정 방법과 채점 절차를 함께 명시한다. 채점자 편향은 §14 부록 채점 코드북으로 차단한다.

### 5.1 M1 — 첫 진입~첫 캠프 진입 성공 시간

- 목표: 30분 이내, 통과 기준 4/5
- 채점: 시작 = `curl install` 실행 시점, 종료 = 캠프 화면 첫 렌더링
- 자동 측정: 화면 녹화 타임스탬프 추출

### 5.2 M2 — 5초 회복 성공률

- 목표: 80%
- 절차: T3 직후 캠프 화면을 5초 응시 → 화면 가림 → 자기 진술
- 정답 정의: (a) 직전 작업 미션명 + (b) 다음 행동 두 항목 모두 일치 시 100%, 한 항목만 일치 시 50%
- 채점자: 2인 독립 채점, Cohen's kappa ≥ 0.7 합의 시 채택

### 5.3 M3 — 화로 5종 어휘 학습 곡선

- 목표: 시험자당 평균 5회 자발적 사용
- 채점 코드북 (§14 부록):
  - 한국어 명("불씨"·"모닥불"·"등불"·"연기"·"장작") 1회 = 1.0점
  - 영문 음차("bulssi"·"modakbul"·"deungbul"·"yeongi"·"jangjak") 1회 = 1.0점
  - 비유 표현("작은 불"·"강한 불"·"끄질 듯한 불") 1회 = 0.5점
  - 의미 외 사용("연기 자욱하다") 0점
- 정규화: 시험자 발화량 1,000단어당 정규화

### 5.4 M4 — 정성 키워드 빈도

- 목표: 긍정:부정 비율 ≥ 3:1
- 긍정 키워드: 예쁘다·차분하다·어렵지 않다·재미있다·다시 열고 싶다
- 부정 키워드: 복잡하다·어렵다·기업적이다·답답하다·다시 안 열겠다
- 채점자: 2인 독립 채점, Cohen's kappa ≥ 0.7
- 추가 후보 키워드는 채점자 합의 후 사전에 추가

### 5.5 M5 — 재진입 의향

- 목표: 1주 후 자발적 재사용, 통과 기준 3/5
- 추적 방법: 자기 보고 설문(시험자 자발적 응답) + 텔레메트리 옵트인 병행
- 텔레메트리 부재 시 자기 보고를 우선
- M5는 보조 지표로 분류. M1~M4 모두 통과한 경우에만 결정적 가중치 부여

## 6. 정성 인터뷰 양식

시험 종료 후 30분 인터뷰. 6종 질문은 필수 골격으로 고정하며, 변형은 후속 질문(probing)에 한정한다.

### 6.1 6종 필수 질문

1. "이 도구를 한 단어로 설명해 보세요"
2. "다시 열고 싶은가요? 왜 그렇게 느끼시나요?"
3. "어디서 헷갈렸나요?"
4. "어디서 즐거우셨나요?"
5. "친구에게 추천할 수 있을까요?"
6. "이 도구가 사라지면 무엇이 가장 아쉬울까요?"

### 6.2 분석 양식

각 질문 응답에 대해:

- 키워드 추출 (긍정·부정·중립 분류)
- 정서 점수 (1~5)
- 인용 가능 발언 발췌 (보고서 게재 가능 여부 사전 동의)

### 6.3 후속 질문 한도

후속 질문은 응답을 명확화하는 목적에만 사용하며, 다음을 금지한다:

- 시험자에게 특정 답변을 유도하는 질문
- "어렵지 않으셨죠?" 같이 긍정 답변을 강요하는 질문
- 시험자의 부정적 발언을 변호하거나 정정하는 질문

## 9. 인센티브와 윤리

### 9.1 인센티브

시험자 1인당 5만원. 지급 일정:

- 시험 종료 직후 50%
- M5 추적 1주 후 50%

진행률 비례 지급:

- T1 완료: 25%
- T2 완료: 50%
- T3 완료: 75%
- T4 완료: 100%

사전 지급은 평가 너그러움 편향 위험으로 회피한다.

### 9.2 동의 절차

- 화면 녹화·음성 녹음·표정 녹화 사전 서면 동의
- 인용 가능 발언의 보고서 게재 사전 동의 (개별 발언 단위)
- 데이터 보관 기간 명시 동의

### 9.3 데이터 보관

- 시험 종료 90일 후 원본 영상·음성 폐기
- 분석 데이터(키워드·점수·익명 발췌)만 익명화 보관
- 시험자 식별 정보(이름·연락처)는 인센티브 지급 후 즉시 폐기

### 9.4 시험자 권리

- 언제든 중단 가능
- 데이터 즉시 삭제 요청 가능
- 부분 데이터 분석 포함 여부는 시험자 선택

### 9.5 부분 데이터 처리

시험 중간에 이탈하는 경우:

- 진행률에 비례하여 인센티브 지급 (§9.1 비례 표)
- 부분 데이터의 분석 포함 여부는 시험자 동의에 따름
- 분석 포함 시 5인 표본 외 보조 자료로 분류

## 10. 시험 보조 도구

### 10.1 5초 회복 측정 절차

1. 시험자가 캠프 URL 입력
2. 화면 로드 완료
3. 5초 응시 (시험자 자가 카운트 또는 채점자 카운트)
4. 화면 가림 (브라우저 탭 가림 또는 노트북 닫음)
5. 자기 진술 시작

### 10.2 인지 부하 측정 (선택)

NASA-TLX 6요소 양식 (정신적·신체적·시간적·노력·성취감·좌절감)을 권장 보조 도구로 분류. **통과 기준에서 분리**하여 시험자 부담을 경감한다. 시험자가 거부하면 생략 가능.

### 10.3 화면 녹화

- macOS: 화면 기록 도구 또는 OBS Studio
- Windows: OBS Studio
- Linux: OBS Studio 또는 SimpleScreenRecorder
- 해상도 1080p 이상, 프레임 30fps 이상

### 10.4 인터뷰 녹음

- Otter.ai 또는 macOS 음성 메모
- STT 정확도 95% 이상 시범 시험 1회로 검증
- 미달 시 수동 전사로 전환

## 11. 본 프로토콜 갱신 규칙

### 11.1 v0.1 → v1.0 갱신 조건

1차 5인 시험 종료 후 14일 이내 다음을 모두 충족 시 v1.0 갱신:

- 5인 모두에게 §5 M1~M5가 측정 가능했음
- 통과 기준이 모집 풀 편향 없이 적용 가능함이 확인됨
- 시범 시험 결과로 30분 기준의 적합성이 확인됨

### 11.2 추가 5인 시험 자동 발동 조건

다음 중 하나에 해당하면 추가 5인 시험을 자동 발동:

- 1차 통과율 분산이 큼 (예: M1 통과율 2/5~3/5 경계선)
- M2~M5 중 하나가 측정 불가능했음
- 채점자 간 신뢰도 Cohen's kappa < 0.7

추가 시험 결과는 1차 결과와 합쳐 10인 표본으로 재산정한다.

### 11.3 v1.0 갱신 후 변경

v1.0 갱신 이후 본 프로토콜 변경은 발주처 + 4인 자문팀 합의 필요.

| 100개 에이전트 동시 운용 욕구 | 있음 / 없음 |
| 페르소나 적합 점수 (자가) | 1~5 |

채택 기준:

- 코딩 경험 0~3년
- 도구 피로도 척도 ≥ 3
- 사전 노출 0회
- OMC 미사용 + 100개 에이전트 욕구 없음
- 페르소나 적합 점수 자가 ≥ 3

### 14.2 M4 정성 키워드 채점

#### 14.2.1 긍정 키워드 사전

- 예쁘다 / 차분하다 / 어렵지 않다 / 재미있다 / 다시 열고 싶다
- 편하다 / 깔끔하다 / 안정적이다 / 직관적이다 / 신뢰가 간다

#### 14.2.2 부정 키워드 사전

- 복잡하다 / 어렵다 / 기업적이다 / 답답하다 / 다시 안 열겠다
- 헷갈린다 / 무섭다 / 부담스럽다 / 압박감 / 시끄럽다

#### 14.2.3 채점 절차

1. 채점자 2인 독립 채점
2. 사전 외 추가 후보 키워드는 채점자 합의 후 추가
3. Cohen's kappa < 0.7 시 채점자 1인 보강
4. 긍정·부정 횟수 합산 후 비율 산정

### 14.3 채점자 신뢰도 검증

- Cohen's kappa 산정: SPSS 또는 Python `sklearn.metrics.cohen_kappa_score`
- 합의 도달 기준: kappa ≥ 0.7
- 미달 시: 채점자 1인 보강하여 3인 합의 채택
