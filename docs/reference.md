## CLI Commands

### `campsite`

Interactive launcher. Presents project and agent selection menu,
then runs sync, lock, agent launch, and cleanup in one flow.

```bash
campsite
```

### `campsite setup`

First-run guided configuration. Sets workspace path, scans for
existing projects, detects installed agents.

```bash
campsite setup
```

### `campsite init [path]`

Bootstrap a new campsite project with template files.

```bash
campsite init ~/projects/my-app
campsite init                      # current directory
```

Creates: `README.md`, `status.md`, `handoff.md`, `decisions.md`,
`.gitignore`, `scripts/bootstrap.sh`, plus `docs/`, `src/`, `tests/`,
`deploy/`, `.campsite/` directories.

Idempotent — does not overwrite existing files.

### `campsite sync`

Compile source-of-truth files into agent-native context files.

```bash
campsite sync                      # all detected agents
campsite sync --adapter=claude     # specific agent only
```

Detects installed agents by checking PATH. Only compiles for agents
whose CLI command is found. Stores hash and sync state after compilation.

### `campsite save`

End-of-session cleanup.

```bash
campsite save
```

Steps: credential scan → change detection warning → delete compiled
files → release lock → store hash.

### `campsite status`

Print current project state summary.

```bash
campsite status
```

Shows: project name, phase, confidence, last-updated, last-agent,
last-device, lock state, integrity (clean/changed/no baseline), next task.

### `campsite peek`

View active session from another terminal.

```bash
campsite peek              # one-shot
campsite peek --live       # refresh every 5 seconds
```

Shows: agent, user, device, terminal, start time, elapsed, task, PID status.

### `campsite go [target] [--sync]`

Jump to a recent project directory.

```bash
campsite go                # interactive: show MRU list
campsite go my-app         # by name
campsite go 1              # by MRU index
campsite go my-app --sync  # jump + auto sync
```

Requires shell wrapper (installed automatically for bash/zsh).
For fish/nushell, see [Shell Wrappers](#shell-wrappers).

### `campsite validate`

Validate project structure, phase, task, and freshness.

```bash
campsite validate
```

Checks: required files exist, valid headers, valid phase value,
non-empty task, freshness (warns if older than `CAMPSITE_STALE_DAYS`).

### `campsite recover`

Clear orphaned locks and stale compiled files after crashes.

```bash
campsite recover
```

Scans all projects in workspace. For each project with a dead-PID lock:
clears lock, cleans compiled files, shows git diff since last sync.

### `campsite dashboard`

Overview of all projects in workspace.

```bash
campsite dashboard
```

Table columns: project name, phase, confidence, lock status, next task.

### `campsite workspace set <path>`

Set the workspace root directory.

```bash
campsite workspace set ~/projects
```

Stored in `~/.campsite/user/config.sh`.

### `campsite --version`

Print installed version.

### `campsite help`

Print usage summary.

## Configuration

### Defaults

File: `~/.campsite/config/defaults.sh`

| Variable | Default | Description |
|---|---|---|
| `CAMPSITE_STALE_DAYS` | `2` | Days before status/handoff is considered stale |
| `CAMPSITE_MAX_WALK_UP` | `20` | Max directory levels to walk up for project detection |
| `CAMPSITE_LOCK_EXPIRY` | `86400` | Lock age in seconds before treating as orphan (fallback) |
| `CAMPSITE_VALID_PHASES` | `discovery building testing reviewing blocked deployed` | Allowed phase values (space-separated) |
| `CAMPSITE_SOURCE_FILES` | `status.md handoff.md` | Files used for hash computation (space-separated) |
| `CAMPSITE_DECISION_COUNT` | `5` | Number of recent decisions in compiled context |
| `CAMPSITE_HEADER_MARKER` | `<!-- Generated by campsite sync. Do not edit directly. -->` | Marker in compiled files |

### User Overrides

File: `~/.campsite/user/config.sh` (never overwritten by updates)

```bash
CAMPSITE_WORKSPACE="$HOME/projects"
CAMPSITE_STALE_DAYS=7
CAMPSITE_VALID_PHASES="discovery building testing reviewing blocked deployed maintenance"
```

### Environment Variables

| Variable | Purpose |
|---|---|
| `CAMPSITE_HOME` | Installation directory (default: `~/.campsite`) |
| `CAMPSITE_WORKSPACE` | Workspace root (overrides config file) |
| `CAMPSITE_SKIP_SECURITY` | Set to `1` to skip credential scanning |

## File Formats

### status.md

Required fields (extracted by campsite):

| Field | Location | Example |
|---|---|---|
| `phase` | `- phase: VALUE` | `building` |
| `confidence` | `- confidence: VALUE` | `medium` |
| `last-updated` | `- last-updated: VALUE` | `2026-03-20` |
| `last-agent` | `- last-agent: VALUE` | `claude` |
| `last-device` | `- last-device: VALUE` | `macbook-pro` |

Must start with `# Status`.

### handoff.md

Required fields:

| Field | Location | Example |
|---|---|---|
| `task` | `- task: VALUE` | `implement checkout` |

Must start with `# Handoff`. Task must be non-empty.

### decisions.md

No required fields. Entries are delimited by `## ` headers.
Last N entries (default: 5) are included in compiled context.

### Lock file (`.campsite/lock`)

```
actor: username
tool: claude
pid: 12345
host: macbook-pro
started-at: 2026-03-20T14:30:00Z
task: implement checkout
terminal: /dev/ttys003
```

### Compiled context file

```markdown
<!-- Generated by campsite sync. Do not edit directly. -->
<!-- Source: status.md, handoff.md, decisions.md, README.md -->

# project-name — Session Context

## Current Status
(status.md content)

## Next Action
(handoff.md content)

## Recent Decisions
(last N decisions from decisions.md)

## Project Overview
(README.md content)

## Session Protocol
(instructions for agent to update files before session end)
```

Safety: campsite refuses to overwrite a file that does not contain
the `campsite sync` marker in its first 5 lines.

## Adapter Specification

### Built-In Adapters

| Name | File | Context File | Command |
|---|---|---|---|
| `claude` | `adapters/claude.sh` | `CLAUDE.md` | `claude` |
| `codex` | `adapters/codex.sh` | `AGENTS.md` | `codex` |
| `cursor` | `adapters/cursor.sh` | `.cursorrules` | `cursor` |
| `copilot` | `adapters/copilot.sh` | `.github/copilot-instructions.md` | `copilot` |
| `gemini` | `adapters/gemini.sh` | `GEMINI.md` | `gemini` |

### Adapter File Format

```
name: my-agent
context-file: .my-agent-context.md
location: project-root
format: markdown
command: my-agent
sections: status handoff decisions readme
```

| Field | Required | Description |
|---|---|---|
| `name` | Yes | Adapter identifier |
| `context-file` | Yes | File to generate in project root |
| `location` | No | Output location (default: `project-root`) |
| `format` | No | Output format (default: `markdown`) |
| `command` | Yes | CLI command to launch the agent |
| `sections` | No | Sections to include (default: `status handoff decisions readme`) |

### Custom Adapter

Create at `~/.campsite/user/adapters/my-agent.sh`.
User adapters take precedence over built-in adapters of the same name.

### Adapter Resolution Order

1. `~/.campsite/user/adapters/NAME.sh` (user override)
2. `~/.campsite/adapters/NAME.sh` (installed)
3. `$CAMPSITE_ROOT/adapters/NAME.sh` (dev mode)

## Shell Wrappers

The `campsite go` command prints a path to stdout. A shell wrapper
is needed to `cd` into that path in the current shell.

### Bash / Zsh

Installed automatically by `install.sh` into `.bashrc` / `.zshrc`.

### Fish

Add to `~/.config/fish/config.fish`:

```fish
function campsite --description "Campsite CLI with cd support"
    if test (count $argv) -gt 0; and test "$argv[1]" = "go"
        set -l target (command campsite $argv)
        if test $status -eq 0; and test -n "$target"; and test -d "$target"
            cd "$target"
            printf "  \033[32m✓\033[0m jumped to: %s\n" "$target"
            return 0
        end
    end
    command campsite $argv
end
```

### Nushell

Add to `~/.config/nushell/config.nu`:

```nu
def campsite [...args] {
    if ($args | length) > 0 and ($args | first) == "go" {
        let target = (^campsite ...$args | str trim)
        if ($target | is-not-empty) and ($target | path exists) {
            cd $target
        }
    } else {
        ^campsite ...$args
    }
}
```

├── lib/                    ← bash libraries

## Project Layout

After `campsite init`:

```
my-app/
├── README.md               ← project description (stable)
├── status.md               ← current state (source of truth, volatile)
├── handoff.md              ← next action (source of truth, volatile)
├── decisions.md            ← decision log (append-only)
├── .gitignore              ← excludes compiled files and .campsite/
├── scripts/bootstrap.sh    ← project setup script
├── docs/
├── src/
├── tests/
├── deploy/
└── .campsite/              ← internal state (gitignored)
    ├── lock                ← session lock
    ├── known-hash          ← hash of source files at last sync
    └── sync-state          ← git HEAD + timestamp at last sync
```

After `campsite sync` (gitignored, deleted by `campsite save`):

```
my-app/
├── CLAUDE.md                           ← for Claude Code
├── AGENTS.md                           ← for OpenAI Codex
├── .cursorrules                        ← for Cursor
├── .github/copilot-instructions.md     ← for GitHub Copilot
└── GEMINI.md                           ← for Gemini CLI
```

## Credential Scanning

On `campsite save`, source files are scanned for patterns matching
credential assignments (API keys, passwords, tokens followed by `=` or `:`
and a value of 8+ characters).

False positive exclusions: comment lines (`#`, `//`, `<!--`),
common phrases (`token limit`, `password policy`, `secret feature`).

Skip scanning: `CAMPSITE_SKIP_SECURITY=1 campsite save`

---

## Lock Behavior

- **Acquisition:** On `campsite sync` (interactive launcher) or manual lock.
- **Release:** On `campsite save` or session cleanup trap.
- **Orphan detection:** PID-based. If the owning process is dead, lock is orphaned.
- **Cross-device:** If lock host differs from current host, PID check is skipped.
  Falls back to age-based expiry (`CAMPSITE_LOCK_EXPIRY`, default 24h).
- **Recovery:** `campsite recover` clears orphaned locks across all projects.

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| "no campsite project found" | Not in a directory with `status.md` + `handoff.md` | `cd` to project or `campsite init` |
| "project locked by ..." | Another session or crashed session holds lock | `campsite recover` |
| "status.md is N days old" | Source files are stale | Edit `status.md`, then `campsite sync` |
| No compiled files after sync | Agent CLI not in PATH | `which claude` to verify; use `--adapter=` flag |
| Hash mismatch warning | Files edited outside campsite session | Accept changes when prompted |
| "invalid phase" | Phase value not in allowed list | Check `CAMPSITE_VALID_PHASES` in config |
| `campsite go` doesn't cd | Shell wrapper not loaded | Open new terminal or source `.bashrc` |

## Uninstall

Remove installation (keep user config):

```bash
rm -rf ~/.campsite/bin ~/.campsite/lib ~/.campsite/adapters \
       ~/.campsite/templates ~/.campsite/config ~/.campsite/VERSION
```

Remove everything:

```bash
rm -rf ~/.campsite
```

Remove shell profile entries (the `# Campsite` block in `.bashrc`/`.zshrc`/`.profile`).

Remove per-project files: `rm -rf .campsite/` in each project directory.
Compiled files are already gitignored and cleaned by `campsite save`.
